# 履修登録システム 使い方ガイド

**対象:** システム管理者・教務担当者  
**システム基盤:** Google Apps Script + Google Spreadsheet

---

## 1. 初期セットアップ

### 1.1 前提条件

- Google Workspace アカウント（教育機関向け推奨）
- Node.js（claspコマンド実行用）
- clasp CLI（`npm install -g @google/clasp`）

### 1.2 GASプロジェクトの準備

```bash
# 1. claspにログイン
clasp login

# 2. .clasp.json を編集し、自校のGASプロジェクトIDを設定
# { "scriptId": "自校のスクリプトID", "rootDir": "." }

# 3. コードをプッシュ
clasp push
```

### 1.3 スプレッドシートの準備

GASプロジェクトに紐づくスプレッドシートに以下のシートを作成：

| シート名 | 役割 | 必須 |
|----------|------|------|
| 科目データ | 科目マスタ | ○ |
| 登録データ | 生徒の履修登録状況 | ○ |
| 教職員データ | 教職員のアカウント・権限 | ○ |
| 設定 | Key-Value形式のシステム設定 | ○ |
| 教科書 | 科目と教科書の対応 | △（未実装） |
| レポート | レポートシステム連携用 | △（未実装） |
| SM設定 | スクーリンググループ分け | △（未実装） |

### 1.4 設定シートの初期値

設定シートは「設定名」「値」の2列構成（Key-Valueストア）：

| 設定名 | 説明 | 推奨値 |
|--------|------|--------|
| 学校名 | システム表示用の学校名 | （自校名） |
| 生徒用ドメイン | 生徒のメールドメイン | 例: `student.school.ac.jp` |
| 本登録許可 | 本登録の開閉 | `不可`（準備完了後に`許可`） |
| 仮登録 | 仮登録の開閉 | `不可`（準備完了後に`許可`） |
| 一時保存 | 一時保存機能の開閉 | `不可` or `許可` |
| 1年最低単位数 | 1年次の最低履修単位数 | 例: `17` |
| 2年最低単位数 | 2年次までの累計最低単位数 | 例: `44` |
| 3年最低単位数 | 卒業に必要な累計単位数 | 例: `74` |
| 卒業要件_理科 | 理科の最低修得単位数 | 例: `4` |
| メンテナンスモード | メンテナンス時のアクセス制限 | 空欄（通常時） |
| ○マーク履修中表示 | ○マークの表示方法 | `有効` |
| 生徒への表示 | 生徒画面の表示形式 | `クラス` or `学籍番号` |

### 1.5 WebApp公開

1. GASエディタを開く（`clasp open`）
2. 「デプロイ」→「新しいデプロイ」
3. 種類: 「ウェブアプリ」
4. 実行ユーザー: 「自分」
5. アクセスユーザー: 「組織内の全員」（Google Workspace の場合）
6. デプロイURLを教職員・生徒に配布

## 2. データ投入

### 2.1 教職員データシート

| カラム | 説明 | 例 |
|--------|------|-----|
| 名前 | 教職員氏名 | `田中 太郎` |
| メールアドレス | Google Workspaceアドレス | `tanaka@school.ac.jp` |
| 権限 | `管理者` / `教務` / 空欄（教員） | `管理者` |
| クラス | 担当クラス | `1-1` |

**重要:** 最初に管理者権限のアカウントを1つ以上登録してください。

### 2.2 科目データシート

| カラム | 説明 | 必須 |
|--------|------|------|
| 学年 | 対象学年 (1/2/3) | ○ |
| セクション | 画面上のグループ名 | ○ |
| 選択可能学年 | 履修可能な学年（カンマ区切り） | ○ |
| 科目名 | 科目の正式名称（一意） | ○ |
| 単位数 | 修得単位数 | ○ |
| 区分 | `必修` / `選択` / `選択必修` | ○ |
| 階段 | 前提科目（段階履修制約） | |
| 同時不可 | 同時履修不可の科目 | |
| 教科 | 教科分類（卒業要件判定用） | |
| 詳細 | 補足説明テキスト | |

### 2.3 登録データシート

生徒を新規登録する場合、最低限以下の基本カラムを用意：

| カラム | 説明 |
|--------|------|
| タイムスタンプ | （自動入力） |
| ステータス | `未提出` / `仮登録` / `本登録` 等 |
| 認証コード | （自動生成） |
| 学籍番号 | 生徒ID |
| 学年 | `0`=入学前, `1`-`3`=在学年 |
| 組 | クラス |
| 番号 | 出席番号 |
| 名前 | 生徒氏名 |
| メールアドレス | 生徒のGoogle Workspaceアドレス |
| 来年度学年 | （自動計算） |

K列以降に科目名カラムが続きます（科目データの科目名と一致させる）。

## 3. 運用フロー

### 3.1 年度開始時の作業

```
1. 科目データの更新（新規科目追加・カリキュラム変更）
2. 教職員データの更新（異動・新任・退職）
3. 新入生データの投入（Excel取込機能を利用）
4. 設定の確認
   - 仮登録: 不可 → 許可（開始時に変更）
   - 本登録許可: 不可
   - メンテナンスモード: 空欄
```

### 3.2 履修登録期間

```
生徒の操作:
  1. WebAppにアクセス → 自動認証（Google Workspace）
  2. 科目を選択（○マークをクリック）
  3. 「一時保存」→ 検討継続
  4. 「仮登録」→ 教員承認待ち

教員の操作:
  1. 教職員専用画面で担当生徒一覧を確認
  2. 生徒の登録内容を確認
  3. 「承認」→ 仮登録承認 / 「差戻し」→ 修正依頼

管理者の操作:
  1. 全生徒の登録状況を一覧で確認
  2. 「本登録許可」を設定で有効化
  3. 本登録許可を生徒個別に付与
```

### 3.3 本登録

```
1. 管理者が設定で「本登録許可」を「許可」に変更
2. 管理者が対象生徒に本登録許可を付与
3. 生徒にQRコード/URLを配布（認証コード付き）
4. 生徒が本登録画面で最終確認 → 「本登録」
```

### 3.4 年度末の作業

```
1. 設定で仮登録・本登録を「不可」に変更
2. 登録データのバックアップ（シートのコピー）
3. 年度移行処理（管理画面から実行）
   - 学年の繰り上げ
   - 登録マークの更新（○→●等）
```

## 5. Excel取込機能

### 5.1 新入生データ取込

**必須列:** `学籍番号`, `名前`  
**任意列:** `組`, `番号`, `学年`（0または1）

1. 管理画面 → 「新入生データ取込」
2. Excelファイルを選択
3. プレビューで確認（重複チェック自動実行）
4. 「取込実行」

### 5.2 テスト結果取込

**必須列:** `学籍番号`, `指導要録科目名`, `テストクリア`（または合否列）, `学年`

1. 管理画面 → 「テスト結果取込」
2. Excelファイルを選択
3. マッチしない科目名の確認
4. 「取込実行」

### 5.3 学籍移動（生徒削除）

**必須列:** `学籍番号`, `学籍移動`  
**学籍移動の値:** `転科`, `転学`, `退学`, `卒業`

1. 管理画面 → 「学籍移動」
2. Excelファイルを選択
3. 対象生徒の確認
4. 「削除実行」

## 6. カスタマイズ

### 6.1 学校情報の変更

`src/00_Config.gs` の `DEFAULTS` セクション：

```javascript
DEFAULTS: Object.freeze({
  SCHOOL_NAME: '通信制高校 履修登録システム',
  // 必要に応じて変更
})
```

または設定シートで動的に変更可能。

### 6.2 科目の区分ルール

科目データシートの「区分」列で以下を指定：
- `必修` — 自動登録・解除不可
- `選択` — 生徒が自由に選択
- `選択必修` — 指定グループ内で必ず選択
- `履修指定` — 指定生徒に自動登録

### 6.3 表示制御

科目データの「表示学籍番号」「非表示学籍番号」で生徒ごとの科目表示を制御：
- **表示学籍番号:** 指定した生徒のみに表示（ホワイトリスト）
- **非表示学籍番号:** 指定した生徒には非表示（ブラックリスト）

### 6.4 卒業要件の設定

設定シートで教科別の最低単位数を設定：
- `卒業要件_理科` = `4` → 理科で4単位以上必要
- 他の教科も同様に `卒業要件_<教科名>` で追加可能

## 7. トラブルシューティング

### よくある問題

| 症状 | 原因 | 対処 |
|------|------|------|
| 「システムエラー」表示 | GASの実行エラー | GASエディタでログを確認 |
| ログインできない | メールアドレスが教職員/登録データに未登録 | シートにメールアドレスを追加 |
| 科目が表示されない | 「選択可能学年」に生徒の学年が含まれない | 科目データシートを確認 |
| 仮登録できない | 設定で「仮登録」が「不可」 | 設定シートで「許可」に変更 |
| 本登録できない | 本登録許可が未付与、または設定で「不可」 | 管理者が個別許可 + 設定変更 |
| データが古い | キャッシュが残っている | ブラウザリロード / 管理画面でキャッシュクリア |

### ログの確認

GASエディタの「実行」メニューからログを確認できます。  
システムは `Logger_` モジュールで操作ログを記録しています。
